## 1. 题目背景与核心目标 (推断/总结)
通过分析测试输入（由连字符分隔的数字区间块，紧跟一个空行，随后是独立的数字序列）和代码逻辑，可以推断本题是关于**一维区间（Intervals）处理**的经典问题。
*   **Part 1** 的目标是处理一系列点查询（Point Queries）：给定一个区间集合和若干个独立的查询点，统计有多少个查询点至少落入了一个区间内。
*   **Part 2** 的目标转移到了区间本身：求所有输入区间的并集所覆盖的**离散整数总数**（即合并重叠区间后求总长度）。

## 2. Part 1 解题思路与算法
* **数据结构选择**: 
  使用 OCaml 原生的元组列表 `(int * int) list` 来存储区间集合。
* **核心算法分析**: 
  采用暴力的**线性扫描搜索**。对于通过 `Scanf` 读入的每一个查询点 `point`，使用 `List.exists` 遍历区间列表，判断是否满足 `a <= point && point <= b`。
  假设有 $R$ 个区间和 $Q$ 个查询点，时间复杂度为 $O(Q \times R)$，空间复杂度为 $O(R)$。由于 Part 1 不需要处理区间之间的相互关系，暴力判定完全能满足需求。
* **难点与细节解析**: 
  Part 1 的主要难点在于**非结构化流式输入的健壮解析**。输入文件由两部分不同格式的数据组成，中间由空行隔开。代码在 `common.ml` 中巧妙地使用 `Scanf.bscanf_opt` 尝试匹配 `%d-%d\n`，当遇到格式不匹配（即空行）时返回 `None`，从而干净利落地跳出区间解析阶段，并将输入流的指针准确停留在点查询数据的起始处。

## 3. Part 2 进阶逻辑与性能考量
* **相比 Part 1 的变化**: 
  Part 2 完全抛弃了单点查询的逻辑，转而要求计算所有区间联合覆盖的整数个数。由于输入的区间可能存在大量重叠和包含关系，不能简单地将所有区间的长度相加。
* **优化策略与详细推导**: 
  算法采用了经典的**区间合并 (Merge Intervals)** 算法。
  1. **排序**: 首先使用 `List.sort` 按照区间的左端点 `a` 进行升序排序。时间复杂度 $O(R \log R)$。
  2. **合并 (Fold)**: 遍历排序后的区间。利用 `List.fold_left` 维护一个已合并区间的累加器列表 `acc`。
     * 设当前要处理的区间为 `(a, b)`，累加器中最顶端（即前一个合并好）的区间为 `(c, d)`。
     * 由于已经按左端点排序，必然有 `c <= a`。
     * **推导交集**: 如果 `a <= d`，说明两个区间有重叠（或包含）。此时合并后的新区间右端点应为 `max(b, d)`。代码中通过精准的分支 `if b <= d then acc else (c, b) :: rest` 实现了不可变数据结构下的高效处理：当新区间完全被包含时（`b <= d`），直接返回原列表 `acc`，利用结构共享（Structural Sharing）避免了重新分配；而在需要延长右端点时，则会分配一个新的元组 `(c, b)` 并构造新的列表节点挂载到 `rest` 上。
     * 如果 `a > d`，说明区间断开，直接将 `(a, b)` 作为独立区间压入累加器 `x :: acc`。
  3. **求和**: 最后遍历合并后相互独立的区间集合，累加计算离散点数 `b - a + 1`。
  该算法通过排序将原本需要 $O(R^2)$ 比较的合并过程降维打击，最终将总时间复杂度优化至 $O(R \log R)$。

## 4. OCaml 实现亮点与语言特性
* **语言特性运用**: 
  * **`[@tail_mod_cons]` (TMC) 编译制导**: 这是本代码最惊艳的底层亮点。在 `common.ml` 的 `input_ranges` 函数中，作者使用了 `x :: input_ranges chan` 这种通常会导致栈溢出的非尾递归写法，但为其标注了 `[@tail_mod_cons]` 属性。这是现代 OCaml (4.14+) 引入的**构造器模尾递归优化 (Tail Modulo Cons)**。编译器会在底层将其自动重写为高效的、不消耗调用栈的指令式循环，既保留了极其优雅的函数式列表构建语法，又获得了尾递归的安全性和性能，是极具专家水准的 OCaml 用法。
  * **高级模式匹配**: 在 `merge_ranges` 函数中，模式匹配 `match acc with | (c, d) :: rest when a <= d -> ...` 完美地将解构（取出栈顶区间）、变量绑定和条件守卫 (`when`) 融合在一行代码中，逻辑极其紧凑。
* **依赖与生态**:
