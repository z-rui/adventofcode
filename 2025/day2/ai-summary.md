## 1. 题目背景与核心目标 (推断/总结)
根据输入数据（如 `11-22,95-115,...`）和代码逻辑推断，本题的目标是处理一系列给定的数值区间，并找出其中具有特定字符串模式的数字。
具体而言，题目定义了一种“无效（invalid）”数字：如果一个数字的字符串形式是由其某个前缀子串重复若干次构成的（例如 `"1212"` 是由 `"12"` 重复两次构成，`"111"` 是由 `"1"` 重复三次构成），则该数字符合条件。核心目标是遍历这些区间，筛选出符合条件的数字并求和。

## 2. Part 1 解题思路与算法
* **数据结构选择**: 
  使用 OCaml 基础的列表 `List` 来存储区间，区间本身用字符串二元组 `(first, last)` 表示。
* **核心算法分析**: 
  作者在代码开头明确标注了 `(* Start with brute force, until it turns out to be too slow. *)`，因此采用的是**纯暴力枚举算法**。
  对于每个区间，使用 `for` 循环遍历其中的每一个整数，将其转换为字符串。Part 1 仅检查字符串是否由某子串重复 **2 次** 构成（`is_invalid 2`）。若是，则将其累加。
* **难点与细节解析**: 
  判断字符串 `s` 是否由子串重复 `n` 次构成的逻辑 `is_invalid n s`：
  1. 首先检查边界条件：字符串长度必须能被 `n` 整除（`len mod n = 0`）。这是避免后续截取子串越界和产生逻辑错误的前置条件。
  2. 截取前缀：取前 `len / n` 个字符作为基础子串。
  3. 字符串构造与比对：将该子串重复 `n` 次生成新字符串，与原字符串 `s` 进行全量比对。这里通过 `Bytes` 模块直接操作内存（`Bytes.blit_string`）来拼接字符串，比传统的字符串连接符 `^` 效率更高。

## 3. Part 2 进阶逻辑与性能考量
* **相比 Part 1 的变化**: 
  Part 1 仅寻找重复恰好 2 次的数字（例如 `"1212"`）。Part 2 放宽了条件，数字的字符串形式可以由某子串重复 **任意 n 次** 构成（其中 n ≥ 2，最大可达字符串本身的长度）。
* **优化策略与详细推导**: 
  算法整体依然维持了暴力枚举的框架。对于生成的每个字符串 `s`，新逻辑 `is_invalid_any s` 会遍历可能的重复次数 `n`（从 2 一直尝试到 `String.length s`）。
  这里的关键细节在于**提前终止（Short-circuiting）**。由于只要找到一种满足条件的 `n` 即可判定该数字符合条件，不需要继续验证后续的 `n`。OCaml 的标准 `for` 循环没有 `break` 语句，因此代码中利用了异常机制来实现短路：一旦内部的 `is_invalid n s` 返回 `true`，就立刻抛出 `Exit` 异常，跳出循环并返回 `true`，从而节省了不必要的字符串重复计算。
* **潜在改进方向**: 
  在判断字符串是否由前缀重复构成时，当前实现通过 `Bytes.create` 并在内存中重复拼接整个字符串来进行比较。一个更高效且无内存分配的改进是采用**原位字符比较**：只需遍历字符串，检查 `s.[i] = s.[i mod (len / n)]` 是否对所有 `i` 成立即可。这将显著减少暴力枚举过程中的对象分配和垃圾回收（GC）压力。
