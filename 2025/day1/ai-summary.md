## 1. 题目背景与核心目标 (推断/总结)
基于对输入数据（如 `L68`, `R48` 等）和核心逻辑的推断，本题模拟了一个在一个大小为 100 的环形轨道（或刻度盘，取值范围 0-99）上的连续移动过程。
初始状态位于刻度 50。输入指令由一个方向（L 代表向左/逆时针减小，R 代表向右/顺时针增加）和一个移动步长组成。
题目的核心目标是追踪这一系列移动指令，并统计在移动过程中触发特定条件（经过或停留在刻度 0，即跨越周期边界）的次数。

## 2. Part 1 解题思路与算法
* **数据结构选择**: 
  核心处理链路使用了 OCaml 标准库中的序列 `Seq.t`。由于只需要单次遍历输入数据，`Seq` 提供了惰性求值（Lazy evaluation）的特性，能够以流水线的方式处理数据，无需将所有中间状态加载到内存中，空间效率极高。
* **核心算法分析**: 
  属于直接模拟算法。通过 `Seq.scan` 函数维护并累加当前的位置状态，初始位置为 `50`。之后使用 `Seq.filter` 过滤出恰好停留在位置 `0` 的状态，并通过 `Seq.length` 统计总次数。
  时间复杂度为 $O(N)$（$N$ 为指令数），空间复杂度为 $O(1)$。
* **难点与细节解析**: 
  这里的难点在于**处理向左移动导致的负数越界问题**。在 OCaml 中，负数取模操作 `%` （或 `mod`）返回的仍然是负数或 0（例如 `-5 mod 100` 的结果是 `-5`，而不是期望的环形坐标 `95`）。
  为了解决这个问题，代码中使用的核心公式是：`(x + y + 100) mod 100`（其中 `x` 是当前坐标，`y` 是位移量，如 `-68`）。
  加上 `100` 的目的是确保被取模的数是一个正数。因为题目中位移量（如 `L68`）虽然向左，但由于它加了 100 之后 `(x + (-68) + 100)` 必然为正数，从而保证了 `mod 100` 运算能正确将坐标映射回 `0-99` 的范围内。这是处理环形坐标系移动的一个经典且极易弄错的细节。

## 3. Part 2 进阶逻辑与性能考量
* **相比 Part 1 的变化**: 
  Part 1 仅统计**最终恰好落点**在 `0` 刻度的次数，而 Part 2 的逻辑转变为统计在移动过程中**跨越（crosses）**边界（0刻度）的总次数。由于单次指令的步长可能大于 100，一次移动可能会跨越边界多次。
* **优化策略与详细推导**: 
  算法从基于状态流的落点记录 (`scan + filter`) 调整为了基于状态聚合的跨越次数计算 (`fold_left`)。
  使用一个二元组 `(crosses, angle)` 作为累加器状态。`crosses` 记录累计跨越边界的次数，`angle` 记录当前所处的绝对坐标（不取模）。
  在每次移动时，位移量 `amount` 累加到 `angle` 得到 `angle'`。这里使用了 OCaml 的 `Int.div` 和 `Int.rem` 来求取完整的周期数 `q` 和余数 `r`。
  
  **难点解析与边界处理推导**：
  这里极其容易出错的地方在于**向左跨越和向右跨越 0 刻度时的计算不对称性**。
  1. **向右或到达正数坐标 (`amount >= 0 || angle' > 0`)**: 这是相对简单的情况。由于从正数开始，每一次除以 100 的商 `q` 就代表着向右跨越了多少个完整的 100 周期。因此新的累计跨越次数就是 `crosses + q`，新的起始点就是余数 `r`。
  2. **向左且到达负数坐标 (`else` 分支)**: 这是最难处理的逻辑。当 `angle'` 为负数时，`div` 和 `rem` 的结果也是负的（或0）。为了准确计算跨越 0 的次数，代码进行了精妙的修正推导：
     - 首先，负向的周期数应该是 `-q`（因为商是负的）。
     - **极易错的边界偏移**: 仅仅 `-q` 是不够的。如果上一次的起点 `angle` 恰好就是 `0`（即在边界上），那么向左走是不会额外算作“跨越”初始边界的，所以增加量就是 `-q + 0`。但是，如果 `angle` 不是 `0`，那么向左哪怕走一小步（例如从 `5` 走到 `-5`，`q` 为 0），也已经跨越了 `0` 刻度一次。因此这种情况下需要额外补偿一个 `1`。这就是 `let q' = -q + if angle = 0 then 0 else 1` 这个关键公式的来源。
     - **余数修正**: 同样，为了让下一次计算的起始坐标重新映射回正数区间 `0-99`，需要对负余数 `r` 进行修正。如果恰好停在 `0`，余数是 `0`；否则，实际的环形位置应该是 `r + 100`。这就是 `let r' = if r = 0 then 0 else r + 100` 的推导过程。
  通过这种极其细致的分支逻辑和补偿推导，算法能够准确 $O(N)$ 地累计出复杂往复运动中的边界跨越次数。
